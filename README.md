# conformal-nutrition: мультимодальная регрессия нутриентов с конформной калибровкой

## Цель проекта

Магистерская работа по предсказанию калорийности и макронутриентов блюда
по его обзорной фотографии и текстовому списку ингредиентов. Базовая модель
учится на парах фото плюс ингредиенты, поверх ее точечных предсказаний
строятся конформные интервалы с гарантированным маржинальным покрытием.
Источник данных — публичный датасет Nutrition5k от Google Research,
лицензия CC BY 4.0. Конечная цель — прикладная: компактный мультимодальный
энкодер с надежными интервалами, который можно запустить на iPhone.

## Как работать на Kaggle

Каждый ноутбук синхронизируется через GitHub-интеграцию Kaggle. Один раз
импортируем ноутбук в Kaggle через File - Import Notebook - Link и
указываем приватный репозиторий conformal-nutrition. Дальше при каждом
открытии ноутбука Kaggle подтягивает актуальную версию из GitHub.
После запуска делаем Save Version - Quick Save, чтобы зафиксировать
output как Kaggle Dataset (важно для ноутбука 01, где мы скачиваем
тысячи изображений).

Перед запуском проверить:
- Settings - Internet on (для скачивания данных в ноутбуке 01).
- Settings - Accelerator - GPU T4 для ноутбуков 02 и 03, CPU для 01.

## Локальная разработка

```bash
git clone git@github.com:<user>/conformal-nutrition.git
cd conformal-nutrition
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
```

Цикл работы: правим ноутбук или код в `src/`, делаем `git push`, на Kaggle
открываем ноутбук - Kaggle подхватывает свежую версию. Ноутбуки на Kaggle
не редактируем напрямую: единственный источник правды — git.

## Артефакты данных

Имена ожидаемых Kaggle Datasets (slug привязывается к пользователю,
конкретные значения добавим после первой публикации):

- `nutrition5k-overhead-rgb-224` — выход ноутбука 01: изображения 224x224,
  parquet с метаданными, разбиения train / calibration / test, EDA.
- `nutrition5k-visual-features` — кэш эмбеддингов визуального энкодера
  (после ноутбука 02).
- `nutrition5k-text-features` — кэш текстовых эмбеддингов
  (после ноутбука 03).
- `nutrition5k-checkpoints` — чекпоинты обученных моделей.

Все последующие ноутбуки подключают предыдущие через `+ Add Data` и не
скачивают данные заново.

## Стиль кода

- Минимализм: каждый файл — одна ответственность.
- Идемпотентность ноутбуков: каждый шаг проверяет наличие итогового артефакта
  на диске и пропускается, если все уже сделано.
- Чистая структура ноутбука: markdown с обоснованием - код - короткий вывод.
- Комментарии только когда смысл неочевиден из кода.
- Никаких эмодзи и эффектов в коде, кроме маркера предупреждения в markdown.
